# Abbott: просмотры по направлению

## Цель

На вкладке «3. Статистика страниц на сайте ABBOTT» добавить четвёртую диаграмму «Просмотры по направлению», чтобы рядом с распределением посетителей было видно распределение просмотров.

## Поведение

- Диаграмма использует уже загруженные строки `page_stats`.
- Для каждого непустого `direction` суммируется поле `pageviews`.
- Данные сортируются по убыванию просмотров и ограничиваются восемью направлениями, как существующая диаграмма «Посетители по направлению».
- Строки без определённого направления не отображаются.
- Диаграмма автоматически учитывает текущий период, поиск и фильтры вкладки, потому что рассчитывается из отфильтрованного набора `pageStatRows`.
- Формат — круговая диаграмма того же типа и с той же цветовой палитрой, что «Посетители по направлению».

## Компоновка

Блок диаграмм меняется с трёх колонок на адаптивную сетку из двух колонок на широком экране. Четыре карточки располагаются в порядке:

1. «Посетители по направлению»;
2. «Просмотры по направлению»;
3. «Посетители по доступу»;
4. «Посетители по типу материала».

На узком экране карточки остаются в одну колонку.

## Реализация

- Вынести агрегирование числовых показателей по направлению в тестируемую функцию модуля статистики страниц.
- Построить `pageViewsDirectionData` из `pageStatRows` и `pageviews`.
- Передать результат в новую `AbbottPieChart`.
- Не менять API, схему БД, загрузчик данных и существующие фильтры.

## Проверка

- Модульный тест подтверждает суммирование просмотров одинакового направления, сортировку, исключение пустых направлений и ограничение топ‑8.
- Существующие тесты страницы проходят без регрессий.
- TypeScript, lint и production build завершаются успешно.
